= 从 VMDK 还原文件和文件夹
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
您可以将一个或多个文件或文件夹从 VMDK 还原到原始 VM 或同一 VMware vCenter 服务器上的其他 VM。支持 Windows 和 Linux 来宾。

.开始之前
* 请查看 link:br-use-vmware-restore-guest-files-and-folders-overview.html["恢复访客文件和文件夹的要求和限制"] 中的先决条件和限制。
* 在从NetApp Backup and Recovery中还原文件和文件夹之前，您需要为来宾虚拟机创建凭据。NetApp Backup and Recovery使用这些凭据在附加虚拟磁盘时对来宾虚拟机进行身份验证。


.关于此任务
文件或文件夹还原性能取决于两个因素：要还原的文件或文件夹的大小；以及要还原的文件或文件夹的数量。如果要还原的数据集大小相同，则还原大量小文件所需的时间可能比还原少量大文件所需的时间要长。

您可以还原到远程 VM（称为跨 VM 还原），但源和目标操作系统必须相同。


IMPORTANT: 一台虚拟机上同时只能运行一个连接或还原操作。您不能在同一台虚拟机上运行并行连接或恢复操作。


IMPORTANT: 使用文件和文件夹还原功能，您可以查看和还原系统和隐藏文件以及查看加密文件。请勿覆盖现有系统文件或将加密文件还原到加密文件夹。在还原操作期间，来宾文件的隐藏、系统和加密属性不会保留在还原的文件中。查看或浏览保留的分区可能会导致错误。

[role="tabbed-block"]
====
.将文件和文件夹还原到原始 VM
--
将来宾文件和文件夹从虚拟磁盘还原到原始（源）虚拟机。

.步骤
. 从NetApp Backup and Recovery菜单中，选择 *Inventory*。
. 选择“虚拟机”菜单。
. 在虚拟机列表中，选择包含要还原的文件的虚拟机的"操作"图标 image:../media/icon-action.png["操作选项"]。
. 选择“恢复文件和文件夹”。
. 选择要从中恢复的快照，然后选择“下一步”。
. 选择要从中恢复的快照位置。如果选择辅助位置，请从列表中选择辅助快照。
. 选择“下一步”。
. 从列表中选择要附加到 VM 的虚拟磁盘，然后选择*下一步*。
. 在 _Guest virtual machine details_ 页面上，执行以下操作：
+
.. 在 *Guest virtual machine details* 部分，通过选择 *Original virtual machine* 将虚拟磁盘附加到原始 VM。
.. 在 *Guest virtual machine credentials* 部分，如果您尚未存储访客虚拟机的凭据，请选择 *Add credentials*，输入 Windows 或 Linux 主机的凭据信息，然后选择 *Add*。
.. 选择用于向客户虚拟机进行身份验证的虚拟机凭据。
.. 选择“下一步”。
+
NetApp Backup and Recovery 将虚拟磁盘连接到虚拟机，并显示所有文件和文件夹，包括隐藏的文件和文件夹。对于 Windows 来宾，它为每个分区（包括系统保留分区）分配一个驱动器号。

+
您可以使用文件浏览器窗格附近的放大镜（搜索）图标来搜索文件和文件夹。不支持模式匹配，但您可以根据名称或扩展名的子集搜索文件或文件夹。



. 选择要还原的文件或文件夹。
+
屏幕右侧窗格中会列出您选择要还原的文件和文件夹。

. 选择“下一步”。
. 在"还原到路径"部分中，输入要将所选文件还原到的来宾路径：
+
** 对于 Windows 来宾，输入 UNC 共享路径：
+
*** IPv4 路径示例： `\\10.60.136.65\c$`
*** IPv6 路径示例： `\\fd20-8b1e-b255-832e--61.ipv6-literal.net\C\restore`


** 对于 Linux 来宾，输入本地文件系统路径或远程来宾地址和路径：
+
*** 本地路径示例： `/home/user/documents/file.txt`
*** IPv4 路径示例： `10.60.136.65:/home/user/restore/`
*** IPv6 路径示例： `fd20-8b1e-b255-832e--61.ipv6-literal.net:/home/user/restore/`




+
如果存在同名文件，您可以选择覆盖或跳过它们。

. 在"后期还原选项"部分，您可以选择在还原完成后通过启用 *还原完成后断开来宾会话* 设置来断开来宾会话。这会分离 VMDK 并卸载数据存储库。这意味着在执行任何其他文件和文件夹还原操作之前，您需要重新连接来宾会话。
. 选择*恢复*。
+
您可以在“作业监控”页面上查看恢复进度。

.将文件和文件夹还原到其他 VM
--
使用代理 VM（与原始 VM 在同一个 vCenter 上的不同 VM）将来宾文件和文件夹从虚拟磁盘还原到另一个虚拟机。

.步骤
. 从NetApp Backup and Recovery菜单中，选择 *Inventory*。
. 选择“虚拟机”菜单。
. 在虚拟机列表中，选择包含要还原的文件的虚拟机的"操作"图标 image:../media/icon-action.png["操作选项"]。
. 选择“恢复文件和文件夹”。
. 选择要从中恢复的快照，然后选择“下一步”。
. 选择要从中恢复的快照位置。如果选择辅助位置，请从列表中选择辅助快照。
. 选择“下一步”。
. 从列表中选择要附加到 VM 的虚拟磁盘，然后选择*下一步*。
. 在 _Guest virtual machine details_ 页面上，执行以下操作：
+
.. 在 *Guest virtual machine details* 部分：
+
... 通过选择*代理虚拟机*将虚拟磁盘连接到代理虚拟机。
... 从列表中选择源 vCenter、数据中心和数据存储库。
... 选择要还原到的虚拟机。所选虚拟机将显示在右侧窗格中。


.. 在 *Guest virtual machine credentials* 部分，如果您尚未存储访客虚拟机的凭据，请选择 *Add credentials*，输入 Windows 或 Linux 主机的凭据信息，然后选择 *Add*。
